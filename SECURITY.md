# Security Policy

## Supported Versions

We generally support the latest minor release on `main` and the latest tagged release.

## Reporting a Vulnerability

Please **do not** report security vulnerabilities via public GitHub issues.

Instead, report privately by email to: **security@future-risk-radar.org**

Please include:

- A description of the vulnerability
- Affected component(s) and version/commit
- Reproduction steps or proof of concept
- Impact assessment (if known)

We will acknowledge receipt within 72 hours and provide status updates as we triage and remediate.

## Disclosure Process

- We triage and reproduce reported issues.
- We develop and validate a fix.
- We coordinate disclosure timing with the reporter when possible.
- We publish a security advisory and remediation guidance.
